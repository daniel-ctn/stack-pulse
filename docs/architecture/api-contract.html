3. Feed cursor format

Keyset pagination — not offset-based.

  1. Server orders by published_at DESC, id DESC.
  2. Cursor encodes JSON { "publishedAt": ISO8601, "id": uuid } as base64url.
  3. Next page: rows where (published_at, id) is strictly older than cursor tuple.
  4. Page size: RELEASE_FEED_PAGE_SIZE = 30; fetches limit + 1 to detect nextCursor.
// encode (server)
Buffer.from(JSON.stringify({ publishedAt, id })).toString("base64url")
