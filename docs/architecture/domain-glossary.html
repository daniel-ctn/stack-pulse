Domain glossary

Status: current · Last verified: 2026-06-04

Terms used consistently in code, UI copy, and documentation.

TermMeaningCode / table
Stack Product concept: the set of technologies a user watches. Often used in UI (“configure stack”, “empty stack”). Implemented as rows in user_tech_preferences
Technology (tech) A single tracked GitHub repository (Next.js, React, or a custom repo). Has name, slug, URL, category. technologies
Registry technology Pre-seeded stack from pnpm db:seed (20 packages). category is framework, library, etc. technologies where category != 'custom'
Custom technology User-added GitHub repo. category = 'custom'. Max 5 per user. addCustomTech
Follow / preference Link between user and technology they want release updates for. user_tech_preferences
Release / release update One version tag from GitHub, plus AI summary and structured fields. Not renamed “article” or “post”. release_updates
Version Git tag name (tag_name from GitHub). Unique per technology. release_updates.version
Signal Semantic tag: breaking, security, deprecation, migration, feature. Stored in release_signals and/or inferred from jsonb arrays. Feed filter signal
Importance AI-assigned priority: low, medium, high, critical. Feed can hide low via “medium+” filter. importance_level enum
Read state Per-user marker that a release was seen in the feed. user_read_releases
Feed Chronological, filterable list of releases. Dashboard for users; API for infinite scroll. getReleaseFeedPage
Public stack page SEO page at /stacks/[slug] showing up to 20 releases for one technology. PUBLIC_STACK_RELEASE_LIMIT = 20
Ingestion / fetch run Batch job that polls GitHub and inserts new summarized releases. release_fetch_runs, cron or custom_repo trigger
Trigger Why a fetch run started: cron (scheduled) or custom_repo (immediate on add). release_fetch_runs.trigger
Digest subscriber Email address stored for a future weekly digest. May be global or tied to stack_slug. digest_subscribers — no emails sent yet
Release advice On-demand AI Q&A about upgrading to a specific release (not stored in DB). /api/release-advice
Slug URL-safe identifier for a technology (nextjs, my-fork). Used in routes and digest. technologies.slug
Cursor Opaque pagination token for feed (base64url JSON of date + id). nextCursor in API
Scope (feed) user — followed techs only; public — all releases in DB. ReleaseFeedScope in release-feed.ts
