3. Cron schedule (source of truth)

{
  "crons": [{ "path": "/api/cron/fetch-releases", "schedule": "0 0,12 * * *" }]
}

Runs at 00:00 and 12:00 UTC daily. Authenticated with Authorization: Bearer <CRON_SECRET> (timing-safe compare in route handler).

Documentation drift Some marketing copy may still say “every 4 hours”. Trust vercel.json and this doc.

5. Production deploy checklist

  1. Connect GitHub repo to Vercel; set all production env vars.
  2. Run pnpm db:migrate against production DATABASE_URL.
  3. If registry empty: pnpm db:seed.
  4. GitHub OAuth callback URL: https://<domain>/api/auth/callback/github.
  5. Set BETTER_AUTH_URL and NEXT_PUBLIC_APP_URL to the canonical HTTPS origin.
  6. Verify cron execution in Vercel dashboard; spot-check release_fetch_runs.

6. Neon + serverless driver

The app uses drizzle-orm/neon-http — each query batch is an HTTP request to Neon, not a long-lived TCP pool. Implications: favor batched reads in one handler (Promise.all on dashboard); avoid chatty N+1 in hot paths; cold starts add latency to first DB call after idle.

8. Self-hosting notes

README targets Vercel + Neon free tier. Any Node host running next start works if you can: (1) provide DATABASE_URL, (2) schedule HTTP cron to /api/cron/fetch-releases with CRON_SECRET, (3) configure GitHub OAuth and public URL env vars.
