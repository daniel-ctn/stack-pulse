StackPulse architecture documentation

Status: current · Last verified: 2026-06-04 · Format: HTML (open in browser)

This folder is the architecture layer for StackPulse: how the system fits together, crosses boundaries, and makes tradeoffs. Feature-level “how to implement X” detail remains in docs/features/*.md; operations runbooks in docs/operations/*.md.

Recommended read order

  1. System overview — context, containers, critical paths
  2. Deployment topology — Vercel, Neon, cron, env by runtime
  3. Database & data architecture — schema, indexes, data flows
  4. Integrations — GitHub, OpenRouter, Better Auth, Vercel Cron
  5. Security — auth matrix, secrets, rate limits, data isolation
  6. API contract — route handlers, server actions, pagination
  7. Frontend architecture — RSC, TanStack Query, nuqs, layering
  8. Observability — what exists, gaps, future ops
  9. Domain glossary — terms used across code and docs
  10. Architecture decision records (ADRs) — why, not what
