System overview

Status: current · Last verified: 2026-06-04

StackPulse is a Next.js 16 application that watches GitHub releases for user-selected developer stacks, summarises them with OpenRouter, and presents a filterable release feed with optional upgrade Q&A. It is non-profit, self-hostable, and has no billing layer.

5. Sequence diagrams

5.1 GitHub sign-in

User → Browser: open /sign-in Browser → StackPulse: GET /api/auth/sign-in/github StackPulse → GitHub: OAuth redirect User → GitHub: authorize GitHub → StackPulse: callback /api/auth/callback/github StackPulse → Neon: upsert user, account, session StackPulse → Browser: Set session cookie, redirect dashboard/onboarding

5.2 Cron ingestion (one technology)

Vercel Cron → API: GET /api/cron/fetch-releases + Bearer CRON_SECRET API → Neon: insert release_fetch_runs (running) API → Neon: list technologies with ≥1 follower loop per tech (chunks of 6 parallel): API → GitHub: latest 5 releases for each new tag: API → OpenRouter: summarizeRelease API → Neon: insert release_updates (on conflict do nothing) API → Neon: update release_fetch_runs (completed)

5.3 Dashboard feed (first page + infinite scroll)

Browser → RSC: GET /dashboard?importance=&signal=... RSC → Neon: getUserTechIds, getReleaseFeedPage (limit 30) RSC → Browser: HTML + initialPage props Browser (client) → API: GET /api/releases?cursor=... (TanStack Query) API → Neon: getReleaseFeedPage with keyset cursor Browser: virtualized list, mark read via server action

6. Non-functional requirements

ConcernCurrent behavior
AvailabilityDepends on Vercel + Neon + third-party APIs; no multi-region app tier
Latency (feed)Keyset pagination, 30 items/page; search uses ILIKE (not FTS)
Latency (ingest)Dominated by OpenRouter per new release; cron bounded by 60s function timeout
ScalabilityCron scans only followed techs; duplicate tech deduped in cron query
ConsistencyNo distributed transactions; preference saves are delete+insert loops
Cost driversOpenRouter calls at ingest; advice calls on demand; Neon storage (raw_release_body)

7. Explicit non-goals (today)

  • Billing / subscriptions (Lemon Squeezy removed in migration 0002)
  • Weekly digest email delivery (signups only)
  • Fetch-run history UI (table exists)
  • PostHog, Sentry, or custom metrics pipeline
  • Automated test suite in CI
  • Row-level security in Postgres (app-layer authorization only)
