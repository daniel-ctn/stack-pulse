Integrations

Status: current · Last verified: 2026-06-04

StackPulse depends on four external integration classes. This document is the cross-cutting reference; feature markdown docs add implementation detail.

1. GitHub

1.1 OAuth (Better Auth)

ItemDetail
PurposeOnly sign-in method
ConfigGITHUB_CLIENT_ID, GITHUB_CLIENT_SECRET
Dev behaviorProvider omitted if env missing (non-production)
Prod behaviorRequired — getAuth() throws if missing
Callback/api/auth/callback/github

1.2 Releases API (ingestion)

ItemDetail
Modulesrc/lib/github.ts — fetchLatestReleases(repoUrl, limit)
Per-tech limitRELEASES_PER_TECH = 5
PublishabilitySkip drafts; require published_at and tag_name
Rate limit60 req/h anonymous; 5000/h with GITHUB_TOKEN Bearer
User-AgentStackPulse on repo verification fetch

1.3 Repo verification (custom stack)

addCustomTech calls GET https://api.github.com/repos/{owner}/{repo} with 8s timeout before insert.

2. OpenRouter (LLM)

ItemDetail
ClientOpenAI SDK → https://openrouter.ai/api/v1 (src/lib/ai.ts)
API keyOPENROUTER_API_KEY (required for summarisation)
ModelOPENROUTER_MODEL or default deepseek/deepseek-chat
RefererNEXT_PUBLIC_APP_URL sent as HTTP referer header

When AI runs

CallTriggerPersisted
summarizeRelease()Cron ingest, custom repo addYes → release_updates (+ model, summarized_at)
adviseOnRelease()POST /api/release-adviceNo — response only
Backfill scriptpnpm releases:backfillUpdates existing rows

Outputs validated with Zod. Untrusted release bodies are treated as prompt data, not instructions.
