ADR-001: Drizzle ORM with Neon HTTP driver

Status: Accepted · Date: 2026-06-04

Context

StackPulse runs on Vercel serverless functions and needs typed SQL access to Postgres. Prisma is not used in this repo. Connection pooling on serverless is historically awkward (too many connections, cold starts).

Decision

  • Use Drizzle ORM with schema in src/db/schema.ts.
  • Use Neon as hosted Postgres.
  • Connect via @neondatabase/serverless and drizzle-orm/neon-http (HTTP per query), not a long-lived node-postgres pool in the app.
  • Migrations via Drizzle Kit in drizzle/.

Consequences

Positive: Lightweight bundle for serverless; schema-as-code; SQL migrations in repo; good TypeScript inference.

Negative: HTTP latency per query batch; no built-in cross-request pool; developers must batch reads and avoid chatty loops.

Neutral: Team must learn Drizzle query API; agents should not assume Prisma patterns.
