ADR-005: HTML format for architecture documentation

Status: Accepted · Date: 2026-06-04

Context

Architecture documentation must be readable by humans in the browser (rich tables, diagrams, navigation) and comprehensive enough for onboarding. Markdown in-repo remains the default for feature/ops docs and agent workflows.

Decision

  • Maintain the architecture layer as static HTML under docs/architecture/.
  • Share styling via docs/architecture/styles.css.
  • Keep database deep-dive at docs/database-architecture.html (linked from hub).
  • Continue feature/ops/agent docs in Markdown for diffability and existing agent read order.

Consequences

Positive: Single-click browser reading; consistent visual design; tables and callouts without MD renderer quirks.

Negative: HTML diffs are noisier than Markdown; agents may prefer MD — hub must link both layers.

Maintenance: Update HTML when architecture changes; trust src/ on conflict.
