ADR-003: Cron ingests only technologies with at least one follower

Status: Accepted · Date: 2026-06-04

Context

The seed registry contains 20 technologies. Continuously fetching all registry repos would burn GitHub API quota and OpenRouter cost for stacks nobody watches. Public SEO pages still need release data for popular stacks.

Decision

  • Scheduled cron loads technologies via INNER JOIN user_tech_preferences (dedupe by tech id).
  • Registry stacks with zero followers are not fetched on cron.
  • Adding a custom repo triggers immediate ingestion for that tech (custom_repo trigger).

Consequences

Positive: Cost and cron duration scale with active user interest, not registry size.

Negative: Public /stacks/[slug] pages stay empty until someone follows that stack (or custom add). SEO for unpopular registry entries depends on first follower.

Operational: Product copy should not imply “we fetch every package in the catalog” on a schedule.
