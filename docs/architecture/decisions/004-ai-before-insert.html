ADR-004: AI summarisation before database insert

Status: Accepted · Date: 2026-06-04

Context

Releases could be stored raw first and summarised asynchronously (queue/worker). StackPulse is serverless-first with no job queue in the repo.

Decision

  • For each new (tech_id, version), call summarizeRelease() before INSERT.
  • Skip insert if summarisation fails (error counted per tech).
  • Use onConflictDoNothing on unique (tech_id, version) for races.
  • Provide pnpm releases:backfill for retroactive summarisation of existing rows.

Consequences

Positive: Feed never shows empty summaries; simpler mental model; no worker infrastructure.

Negative: Ingestion latency tied to LLM; cron more likely to hit 60s timeout when many new releases land at once; OpenRouter outage blocks new rows entirely.

Future: A queue would supersede this ADR if ingest volume or timeout pressure grows.
