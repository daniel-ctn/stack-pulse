ADR-002: Better Auth for GitHub OAuth

Status: Accepted · Date: 2026-06-04

Context

StackPulse needs session-based auth with GitHub as the only identity provider. The app already uses Drizzle; auth tables should live in the same database without a separate auth SaaS (no Clerk/Supabase in scope).

Decision

  • Use Better Auth with drizzleAdapter mapping to users, accounts, sessions, verifications.
  • Expose handler at /api/auth/[...all].
  • GitHub OAuth optional in dev when env vars missing; required in production.
  • Optional @better-auth/infra dash plugin when BETTER_AUTH_API_KEY is set.

Consequences

Positive: First-party session cookies; no per-MAU auth vendor; schema under our migrations.

Negative: We own session security (BETTER_AUTH_SECRET, trusted origins). Must keep adapter schema in sync with library upgrades.

Neutral: Sign-up route redirects to sign-in — single provider flow.
